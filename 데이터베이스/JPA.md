# JPA
**자바 진영에서 ORM을 위해 만든 표준 명세(인터페이스)**

## JPA 사용하는 이유
반복적인 CRUD SQL 작성을 줄여 생산성을 높이고, 객체와 테이블의 패러다임 불일치를 해결해 객체지향적으로 개발할 수 있게 해줘서

## 영속성 컨텍스트
엔티티를 영구 저장하는 환경. 애플리케이션과 DB 사이에서 엔티티를 보관하고 관리하는 논리적인 중간 저장 공간

### 영속성 컨텍스트가 주는 이점
- 1차 캐시

        같은 트랜잭션 안에서 같은 엔티티를 조회하면 DB 안 가고 캐시에서 꺼냄
- 동일성 보장

        같은 트랜잭션 안에서, 같은 id로 조회한 엔티티는 항상 완전히 똑같은 객체(인스턴스)임을 보장
-  쓰기 지연
            
        persist 해도 INSERT 바로 안 나가고, 쿼리를 모아뒀다가 flush/commit 때 한 번에 전송

- 변경 감지

        엔티티를 영속 상태로 만들 때 저장해두고, 커밋 시점에 현재 엔티티와 비교해서 달라졌으면 UPDATE 자동 생성

- 지연 로딩

        연관 엔티티를 실제로 사용하는 시점에 쿼리를 날려서 가져옴

## JPA 생명주기
엔티티가 영속성 컨텍스트와 어떤 관계인지에 따라 4가지 상태를 가진다

-  비영속

        영속성 컨텍스트와 전혀 관계없는 상태

- 영속

        영속성 컨텍스트에 저장되어 관리되는 상태

- 준영속

        영속 상태였다가 영속성 컨텍스트에서 분리된 상태

- 삭제

        엔티티를 삭제하기로 표시한 상태
