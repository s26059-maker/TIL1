# 성능 개선 기초
## 수직 확장과 수평 확장
### 수직 확장
: ` 자원을 증가시키는 것(CPU,메모리,디스크)`

장점 : 즉각적인 효과<br>
단점 : 비용 문제

### 수평 확장
: ` 서버를 늘리는 것 `

<strong>로드 밸런서</strong> : 서버가 2대 이상일 때 트래픽을 서버에 고르게 배분하는 역할

### 주의 할 점
실제 병목 지점이 어디인지 파악하는게 중요하다
<br> 무턱대고 서버를 추가해서는 안된다.

## DB 커넥션 풀

### 의미

`DB에 연결된 커넥션을 미리 생성하고 보관한다`

### 사용 이유
` DB를 사용할 때 다음 3단계를 거친다 `
- DB 연결
- 쿼리 실행
- 연결 종료

<strong>이 때 DB를 연결하고 종료하는데 시간이 많이 쓰이는데</strong> 
<br> 이 문제를 피하기 위해서 <strong> DB 커넥션 풀</strong>을 사용한다

### DB연결을 끊지 않고  필요 할 때 커넥션을 가져와 응답시간이 줄어든다

트래픽이 적은 시간에는 커넥션 풀을 ` 최소 `로,<br> 트래픽이 높은 시간에는 `최대` 크기로 유지하며 필요한 만큼 유지할 수 있음

## 서버 캐시
` (키, 값)으로 저장하는 형태의 데이터 저장소 `

### 종류
<strong>로컬 캐시</strong>: 서버 프로세스와 동일한 메모리를 캐시 저장소로 사용<br>
<strong>리모트 캐시</strong>: 별도 프로세스를 캐시 저장소로 사용

`로컬 캐시`는 빠르지만 데이터 크기에 제한이 있다는 단점이 있다<br>
`리모트 캐시는` 크기를 유연히 확장 할 수 있지만, 속도가 느려지며 시스템 구조가 복잡해진다

## 응답 데이터 압축

응답 시간에는 데이터 전송 시간이 포함되는데 전송 시간은 이 2가지 영향을 받는다
- 네트워크 속도
- 전송 데이터 크기

서버가 네트워크의 속도를 제어할 수 없지만 데이터 크기는 제어할 수 있다.
이때 사용하는 방법이 `응답 데이터 압축`이다

### 클라이언트 캐시
응답 데이터를 클라이언트에 저장해두고, 같은 요청 시 서버까지 안 가고 재사용하는 방식

### CDN
전 세계에 분산된 서버에 콘텐츠를 캐싱해두고, 사용자와 물리적으로 가까운 서버에서 응답하게 하는 시스템
