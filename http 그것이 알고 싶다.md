# HTTP?
## HTTP란...
**HTTP (HyperText Transfer Protocol)** 는
웹에서 클라이언트(브라우저) 와 서버가 데이터를 주고받기 위한 통신 규칙(프로토콜)이다
## HTTP Method
클라이언트가 서버에게 무엇을 하고 싶은지를 표현하는 방식이다

* GET : 리소스(자원) 조회

* POST : 데이터 추가, 등록

* PUT : 리소스 대체, 수정 혹은 생성

* DELETE : 리소스 삭제

* PATCH : 리소스 부분 병경

* HEAD : GET과 동일(body 부분 제외)

* OPTION : 통신 옵션 확인

* CONNECT : 자원으로 식별되는 서버에 연결 요청

## HTTP Status Code
### 상태 코드란?
서버가 요청 결과를 숫자로 알려주는 것이다.

* 1XX : 정보제공

* 2XX : 성공

    * 200 : 성공
    * 201 : 생성됨
    * 202 : 허용됨
* 3XX : 리다이렉션(필요한 추가 동작)

    * 301 : 영구 이동
    * 303 : 다른 위치 보기
    * 307 : 임시 리다이렉션 요청 필요
* 4XX : 클라이언트 에러

    * 400 : 잘못된 요청
    * 401 : 권한 없음
    * 403 : 금지됨
    * 404 : 찾을 수 없음
* 5XX : 서버 에러

    * 500 : 내부 서버 오류
    * 501 : 구현되지 않음
    * 502 : 불량 게이트웨이

    ## HTTP의 무상태성
### HTTP 무상태성 특징
HTTP는 이전 요청을 기억하지 않는 특징이 있습니다.

* 요청 A → 처리
* 요청 B → A와 관련 없음 (기억 못함)

 **특징**

* 각 요청은 독립적
* 서버는 클라이언트 상태를 저장하지 않음

 **장점**

* 서버 구조 단순
* 확장성 좋음 (대규모 서비스에 유리)

 **단점**

* 로그인 상태 유지 어려움
