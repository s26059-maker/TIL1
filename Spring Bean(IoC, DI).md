# Bean?

: Spring이 관리하는 객체

예)<br>
Spring이 만든 캐릭터(객체)

- 너가 직접 안 만듦
- 게임이 알아서 만들어둠

# IoC

: 객체 생성은 Spring이 한다

예)<br>
내가 만들던 걸 이제 Spring이 대신 만듦

- 원래는 내가 캐릭터 생성
- 지금은 게임이 캐릭터 생성

# DI

: 필요한 객체를 Spring이 넣어준다

예)<br>
필요한 장비를 자동으로 끼워줌

- 칼 장착
- 방패 장착


<br>

# 정리

- Spring이 캐릭터 만들고 (IoC)
- 그걸 Bean으로 관리하고
- 필요한 장비까지 넣어줌 (DI)
