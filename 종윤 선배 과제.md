# Spring Security & JWT / Session / Cookie 핵심 정리

## 1. Spring Security

Spring Security는 **Spring에서 인증(Authentication)과 인가(Authorization)를 담당하는 보안 프레임워크**이다.

### 주요 기능
- 로그인(인증)
- 권한 관리(인가)
- 비밀번호 암호화
- 세션 및 JWT 인증 지원
- CSRF, CORS 보안 기능 제공

---

## 2. Cookie / Session / JWT

### Cookie
- 브라우저에 데이터를 저장하는 공간
- 요청 시 자동으로 서버에 전송
- Session ID나 JWT를 저장하는 데 사용

### Session
- 로그인 정보를 **서버**가 저장
- 브라우저는 Session ID만 Cookie에 저장
- 보안성이 높지만 서버 메모리를 사용

### JWT
- 로그인 정보를 **토큰**에 저장
- 요청 시 `Authorization: Bearer {JWT}` 헤더로 전송
- 서버에 세션을 저장하지 않아 REST API에 적합

| 구분 | Session | JWT |
|------|---------|-----|
| 저장 위치 | 서버 | 토큰 |
| 서버 메모리 | O | X |
| REST API | △ | O |
| 로그아웃 | 쉬움 | 상대적으로 어려움 |

---

## 3. Spring Security 주요 구성 요소

- **SecurityFilterChain** : 보안 설정
- **UserDetails** : 사용자 정보
- **UserDetailsService** : 사용자 조회
- **Authentication** : 인증 정보
- **PasswordEncoder** : 비밀번호 암호화
- **SecurityContext** : 로그인 정보 저장

---

## 4. 비밀번호 암호화

Spring Security에서는 **BCryptPasswordEncoder**를 사용한다.

### 특징
- 비밀번호를 암호화하여 저장
- 같은 비밀번호도 매번 다른 암호문 생성(Salt)
- 복호화 불가
- 로그인 시 `matches()`로 비교
